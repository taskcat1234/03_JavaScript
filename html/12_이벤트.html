Event(이벤트)

    * Event : 사건(행동, 동작, 행위), 사고, 일어난 일, 행사
    -> 브라우저에서 할 수 있는 동작

    ex) click, keydown, keypress, keyup,
    mouseover, mouseout, mouseon, mouselaeve,
    change, submit, focus, blur, drag .........기타등등

    * Event Listener(이벤트 리스너)

      - 브라우저에서 발생하는 이벤트를 감지

      - 이벤트가 감지 될 떄 까지 대기

      - 이벤트가 감지 되었을 때 연결된 기능(함수) 수행

      - 이벤트 리스너는 요소 단위로 추가 가능

      - 이벤트 리스너 추가 방법
        1) on이벤트명
          ex) onclick, onkeyup, onchange
        2) 요소.addEventListener("이벤트명",수행할 함수);
    * Event Handler(이벤트 핸들러, 이벤트 처리기)
      - 이벤트가 감지 되었을 떄
        수행할 동작을 작성하는 함수

인라인 이벤트 모델(inline)

    - HTML 요소에 직접
    이벤트 리스너 = "이벤트 핸들러"
    형식으로 작성하는 방법

    - 최근에는 사용 빈도 낮아짐(가끔 사용)

    - 인라인 이벤트 모델 작성 시
      이벤트 핸들러 () 안에 this라는 구문을 작성하면
      이벤트가 핸들러 함수로
      이벤트가 발생된 요소가 전달된다!
인라인 이벤트 모델 확인

고전 이벤트 모델

    - HTML 요소에 이벤트 관련 코드를 직접 작성하지 않고

      JS에서 DOM을 이용해
      이벤트가 처리되기 원하는 요소가
      가지고 있는 고유 이벤트 리스너에
      이벤트 핸들러를 대입하는 방법

    * 단점 : 하나의 요소에 동일한 이벤트 리스너/핸들러 작성 불가
      (마지막으로 대입된 이벤트 핸들러로 덮어쓰기 된다!)

표준 이벤트 모델(요소.addEventListener())

    - W3C (웹 표준 재정 단체)에서
      공식적으로 지정한 이벤트 모델
    
    - 고전 이벤트 모델의 "단점"
      -> 이벤트 리스너가 종류별로 1개
    - 표준 이벤트 모델의 "장점"
      -> 이벤트 리스너를 원하는 만큼 추가할 수 있다!
    [ 작성법 ]
    요소.addEventListener("이벤트종류", 이벤트핸들러);

클릭할 수록 투명 +클릭 횟수 증가
